## 1. 项目概述与目标

### 1.1 产品一句话定位

**MuseCut = 帮普通用户把真实情绪、影像、文字和声音组织成作品的 AI 情感创作平台。**

与"输入 prompt 直接生成音乐"的通用 AI 作曲工具不同，MuseCut 的核心差异在于 **AI 导演 + Music Timeline** 机制：系统先理解用户想表达的情绪，再分析视频的时长、场景变化、关键事件、字幕节点、人声区间和情绪曲线，自动生成可编辑的声音时间轴，最后输出带配乐、Beat、SFX 和人声避让的短视频成片。

MVP 的产品北极星是：**用户不需要懂剪辑和编曲，也能把一段想念、告别、亲情、校园记忆或日常旅行做成完整作品。**

### 1.2 MVP 范围边界

| 维度 | MVP 范围（必须做） | 超出范围（后续迭代） |
|------|-------------------|---------------------|
| 视频输入 | 手动上传短视频（≤60s），填写表达意图；关键节点标注作为高级选项 | 自动 OCR 字幕识别、自动语音识别、长视频批量拆条 |
| AI 编排 | 默认自动生成 Music Timeline JSON，支持 6 种表达方向；无 API key 时默认模板降级 | 多模态 Transformer 联合建模、本地 DeepSeek/Qwen provider |
| 声音生成 | ACE-Step-1.5（服务端 BGM）+ Tone.js（前端 Beat/SFX/Ducking） | 完整 DAW 级混音、专业母带 |
| 导出 | 默认导出 MP4 成片视频；WAV/FLAC/MP3/JSON 作为补充素材 | 多分辨率转码、字幕烧录、封面自动生成 |
| 社区 | 类朋友圈/微博的信息流，支持匿名或实名发布、浏览、点赞、评论、收藏 | 关注/私信/推送/排行榜 |
| 教学 | 静态内容页 + 模板示例教程 | 交互式 AI 助教对话 |

## 2. MVP 产品架构 —— 5 层架构

MuseCut MVP 采用五层架构设计，每一层独立职责、层间通过标准化 JSON 协议通信，可独立测试和替换。

```
┌──────────────────────────────────────────────────────────────┐
│                    【第 1 层】前端交互层                         │
│  表达输入 → 素材上传 → AI 自动编排 → 少量微调 → 预览导出       │
│  技术栈: React 18 + Vite 5 + TypeScript 5 + Tone.js 15        │
│  核心价值: 用户直接操作界面，承载全部创作交互                    │
├──────────────────────────────────────────────────────────────┤
│                    【第 2 层】视频分析层                         │
│  时长 / 关键点 / 剪辑点 / 字幕 / 人声区域 → video_profile.json │
│  技术栈: ffmpeg 帧差检测 + 用户手动标记                         │
│  核心价值: 将原始视频转化为 Agent 能理解的结构化输入              │
├──────────────────────────────────────────────────────────────┤
│                    【第 3 层】AI 编排层                          │
│  输出: BPM / 调式 / 情绪曲线 / 音效节点 → Music Timeline JSON  │
│  技术栈: GLM-4-Flash / 可替换本地 LLM 结构化 JSON 生成               │
│  核心价值: 项目核心创新——先理解视频，再编排声音，不是盲目生成     │
├──────────────────────────────────────────────────────────────┤
│                    【第 4 层】声音生成层                         │
│  BGM (ACE-Step-1.5) + Beat/SFX (Tone.js) + Ducking           │
│  技术栈: ACE-Step 服务端高质量生成 + Tone.js 浏览器端低延迟触发  │
│  核心价值: 可插拔声音引擎——壁垒在编排，不在绑定某个音乐模型       │
├──────────────────────────────────────────────────────────────┤
│                    【第 5 层】导出层                            │
│  MP4 成片导出 + WAV / FLAC / MP3 / JSON 补充导出               │
│  技术栈: ffmpeg 格式转换 + 多轨混音                             │
│  核心价值: 输出接近真实短视频制作流程的产物                       │
└──────────────────────────────────────────────────────────────┘
```

### 2.2 视频分析层

**输入→处理→输出完整链路**：

```
视频文件 (mp4/mov/avi)
    │
    ├─→ [步骤 1: ffmpeg 元数据提取]
    │      命令: ffprobe -v quiet -print_format json -show_format -show_streams video.mp4
    │      输出: duration, fps, width, height, codec, bitrate
    │      → 写入 metadata 字段
    │
    ├─→ [步骤 2: ffmpeg 抽帧]
    │      命令: ffmpeg -i video.mp4 -vf fps=2 frames/frame_%04d.jpg
    │      说明: 每秒抽 2 帧（每 0.5s 一帧），临时存于 temp/ 目录
    │      → 产生帧序列 JPEG 文件
    │
    ├─→ [步骤 3: 帧差计算]
    │      算法: 
    │        for i in 1..len(frames)-1:
    │          diff = mean_absolute_difference(frames[i-1], frames[i])
    │          if diff > 30:  → hard_cut (置信度 = diff/100)
    │          if diff > 15:  → dissolve  (置信度 = diff/100)
    │      输出: scene_change_candidates[] — 自动检测的疑似转场时间点
    │      → 作为"建议"展示给用户（可接受/拒绝/修改）
    │
    ├─→ [步骤 4: 用户手动标记]
    │      前端交互:
    │        - key_events[]: 在时间轴上点击添加标记 → 选择类型+标签+重要性
    │        - voice_regions[]: 拖拽起止手柄标记人声区间
    │        - caption_events[]: 点击时间点 + 输入字幕文字+样式
    │      → 用户手动补充 AI 无法自动识别的内容
    │
    └─→ [步骤 5: 拼装 video_profile.json]
           整合: 步骤1元数据 + 步骤3自动检测 + 步骤4用户标记 + 步骤4视频类型+描述
           输出: 完整 video_profile.json
           存储: 写入 video_projects 数据库记录
```

### 2.3 AI 编排层

**定位**：核心创新模块。接收 `video_profile.json`，调用 GLM-4-Flash（后续可替换为本地 DeepSeek/Qwen/Ollama）输出严格 JSON 格式的 Music Timeline。

```
video_profile.json
    │
    ├─→ [步骤 1: 构建 System Prompt]
    │       注入:
```

### 2.4 声音生成层

**定位**：将 Music Timeline JSON 转化为可听见的声音。采用 **"服务端高质量 BGM + 浏览器端低延迟 Beat/SFX/Ducking"** 的分层策略。

```typescript
  async init(): Promise<void> {

  // 预加载本地音效文件
  private async preloadSFX(): Promise<void> {
    const sfxFiles = [
      { name: 'whoosh', url: '/sfx/whoosh_short.wav' },
      { name: 'hit',    url: '/sfx/hit_impact.wav' },
      // ... 更多音效
    ];
    for (const sfx of sfxFiles) {
      const buffer = await new Tone.Buffer(sfx.url).loaded;
      this.sfxBuffers.set(sfx.name, buffer.get() as AudioBuffer);
    }
  }

  // 核心方法：根据 Music Timeline JSON 编排所有声音事件
  scheduleTimeline(timeline: MusicTimeline): void {
    Tone.Transport.cancel();  // 清除之前的调度

    for (const seg of timeline.timeline) {
      // === 调度 Beat ===
      if (seg.beat_pattern) {
        this.scheduleBeat(seg.beat_pattern, seg.start, seg.end, timeline.bpm);
      }

      // === 调度 SFX ===
      if (seg.sfx) {
        this.scheduleSFX(seg.sfx, seg.start);
      }

      // === 调度 Fade ===
      if (seg.fade === 'fade_in') {
        this.bgmVolume.volume.rampTo(Tone.gainToDb(seg.volume), 0.5, seg.start);
      }
    }

    // === 调度 Ducking ===
    for (const d of timeline.ducking_schedule) {
      this.scheduleDucking(d);
    }
  }

  // 按 beat_pattern 在 [start, end] 区间内按 BPM 定时触发鼓点
  private scheduleBeat(pattern: string, start: number, end: number, bpm: number): void {
    const beatInterval = 60 / bpm; // 每拍秒数
    for (let t = start; t < end; t += beatInterval) {
      Tone.Transport.schedule((time) => {
        this.kick.triggerAttackRelease('C2', '8n', time);
        if (t % (beatInterval * 2) < 0.01) {
          this.snare.triggerAttackRelease('8n', time);
        }
      }, t);
    }
  }

  // 在指定时间触发音效
  private scheduleSFX(sfx: SFXNode, startTime: number): void {
    const buffer = this.sfxBuffers.get(sfx.type);
    if (!buffer) return;
    const player = new Tone.Player(buffer).connect(this.sfxVolume);
    Tone.Transport.schedule((time) => {
      player.start(time);
    }, startTime);
  }

  // 人声区间 ducking
  private scheduleDucking(d: DuckingNode): void {
    // 进入人声：降低 bgmVolume
    Tone.Transport.schedule(() => {
      this.bgmVolume.volume.rampTo(-d.reduce_db, 0.1);
    }, d.start);
    // 离开人声：恢复 bgmVolume
    Tone.Transport.schedule(() => {
      this.bgmVolume.volume.rampTo(0, 0.3);
    }, d.end);
  }

  play(): void { Tone.Transport.start(); }
  stop(): void { Tone.Transport.stop(); }
  setBPM(bpm: number): void { Tone.Transport.bpm.value = bpm; }
  dispose(): void { /* 清理所有 Tone.js 对象 */ }
}
```

**音效素材包清单**（10 个，需预先收集，来源为 freesound.org / pixabay 免费授权）：

### 2.5 导出层

| P2 可选 | 合成视频 | 原始视频 + 音频合并 | `ffmpeg -i video -i audio -c:v copy out.mp4` |
| P2 可选 | 项目文件 | 完整项目状态（可重新编辑） | JSON + 素材文件打包 ZIP |

**导出核心流程**：

```

  步骤 1: 加载 BGM 音频（ACE-Step 生成结果）
  步骤 2: 用 Tone.js 离线渲染 Beat + SFX 轨道为 WAV（Tone.OfflineContext）

  ↓
```

**目标**：根据 `video_profile.json` 调用 Timeline LLM 生成可编辑的 Music Timeline JSON。当前代码使用 GLM-4-Flash，后续可替换为本地 DeepSeek/Qwen。这是项目**核心创新**，决定 MuseCut 与普通 AI 作曲工具的本质差异。

   - 示例至少包含 1 个治愈 vlog + 1 个高燃剪辑的完整 video_profile → timeline 映射

### 3.3 Module 3：可控声音生成与多风格改编

**目标**：在不依赖昂贵商业模型的情况下，用 ACE-Step-1.5（服务端）+ Tone.js（浏览器端）实现稳定可演示的 BGM/Beat/SFX 生成，支持 6 种风格。

**核心产出**：

| 产出物 | 具体文件 | 描述 |
|--------|---------|------|
| ACE-Step 集成服务 | `backend/app/services/acestep_service.py` | 提交/轮询/下载/拼接 BGM 音频 |
| Tone.js 引擎 | `frontend/src/engine/musecutEngine.ts` | 单例引擎，四轨道管理 |
| 音效素材包 | `frontend/public/sfx/` | 10 个免费授权音效 |
| Beat 预设 | `frontend/src/engine/beatPresets.ts` | 4 种节奏模式定义 |
| 风格模板配置 | `frontend/src/config/styleTemplates.ts` | 6 种风格参数预设 |
| BGM 生成 API | `backend/app/routers/generation.py` | `POST /api/generate/bgm` |

**具体执行步骤**：

1. **ACE-Step-1.5 环境部署**
   - 安装依赖：`cd ACE-Step-1.5 && uv sync`
   - 配置 `.env`：turbo 模型 + 0.6B LM
   - 启动 API：`uv run acestep-api`，验证 `/health` 可用
   - 测试生成：提交一个测试 prompt → 轮询 → 下载音频 → 验证

2. **ACE-Step 后端封装 Service**
   - `submit_task(prompt, duration, inference_steps=8)` → 返回 task_id
   - `poll_task(task_id, timeout=120)` → 轮询直到完成，返回 result
   - `download_audio(file_path)` → 下载到 MuseCut 本地存储
   - `generate_bgm_for_timeline(timeline)` → 核心方法
     - 遍历 timeline.timeline 每个 segment
     - 用 `segment.caption` 作为 prompt 提交 ACE-Step
     - 用 `segment.end - segment.start` 作为 duration
     - 收集所有段音频 → ffmpeg concat 拼接为完整 BGM
   - 返回完整 BGM 音频 URL

3. **Tone.js 前端引擎实现**（1.5 天，详见 2.4 节代码设计）
   - 单例 `MuseCutAudioEngine`
   - 初始化：`Tone.start()` + 预加载所有 SFX buffer
   - BGM 播放：`Tone.Player` 加载服务端 ACE-Step 音频 URL
   - Beat 合成：`MembraneSynth`(kick) + `NoiseSynth`(snare) + `MetalSynth`(hat)
   - SFX 触发：预加载 AudioBuffer Map → 按节点 schedule
   - Ducking：`Volume.rampTo()` 平滑降音量
   - 核心调度方法：`scheduleTimeline(timeline)` → 一次性编排所有声音事件

4. **音效素材收集**
   - 从 freesound.org / pixabay 搜索和下载 10 个免费授权音效
   - 放入 `frontend/public/sfx/` 目录
   - 确保格式统一为 WAV（Tone.js 兼容性最好）

5. **Beat 预设定义**
   - 4 种节奏模式的数据结构：simple_kick_snare / energetic_beat / lofi_beat / trap_hats
   - 前端按 BPM 映射为 16 分音符步进序列

6. **风格模板配置**
   - 6 种风格的金参数预设（BPM 范围/默认乐器/和弦模板/情绪映射）

### 3.4 Module 4：多轨后处理与导出

**目标**：让输出更接近真实短视频制作流程——不是单条 BGM，而是四轨分层声音方案。

**核心产出**：

| 产出物 | 具体文件 | 描述 |
|--------|---------|------|
| 导出服务 | `backend/app/services/exporter.py` | 多轨混音 + 格式转换 + 可选视频合成 |
| 导出 API | `backend/app/routers/export.py` | `POST /api/export/audio` |

| 前端混音面板 | `frontend/src/components/preview/MultiTrackMixer.tsx` | 四轨音量滑块 |

**具体执行步骤**：

   - 输入：project_id + timeline + 各轨道音频路径
   - BGM 轨道：从 ACE-Step 生成结果加载
   - Beat 轨道：根据 timeline beat_pattern 用 pydub 生成简单 kick/snare
   - SFX 轨道：加载本地音效，按 timeline sfx 节点放置
   - ffmpeg amix 混音三轨 → 输出 WAV
   - Ducking 处理：在人声区间降低 BGM 音量（ffmpeg volume filter）

   - WAV → FLAC（无损压缩）

3. **前端导出页**
   - 格式选择：WAV / FLAC / MP3 / JSON（配乐方案）
   - 导出进度轮询
   - 下载按钮

4. **前端混音面板**
   - 四轨独立音量滑块（BGM/Beat/SFX/Master）
   - 预览播放
   - "导出"按钮跳转到导出页

5. **可选视频合成**
   - `ffmpeg -i original_video.mp4 -i mixed_audio.wav -c:v copy -c:a aac final.mp4`

**四轨道交付清单**：

| 轨道 | MVP 内容 | 生成方式 |
|------|---------|---------|
| BGM Track | 背景音乐和氛围铺底 | ACE-Step-1.5 text2music 服务端生成 |
| Beat Track | 卡点节奏、鼓点强化 | pydub 简单合成 kick/snare/hat |
| SFX Track | 转场、字幕强调、高潮进入 | 本地音效包按节点放置 |
| Voice-aware Ducking | 人声出现时压低 BGM | ffmpeg volume filter 动态处理 |

### 4.2 架构模式

- **前端**：页面级路由 + 页面内聚实现为主；当前尚未拆出规划中的 upload/editor/preview 等组件目录，短期优先保证演示闭环，后续再组件化
- **后端**：Router → Service → Model 三层架构（参考失恋广场已验证的模式）
- **ACE-Step 通信**：MuseCut 后端通过 HTTP 调用 ACE-Step REST API（localhost:8001）
- **声音分层**：BGM=服务端 ACE-Step 高质量生成 / Beat+SFX+Ducking=浏览器端 Tone.js 低延迟实时触发；服务端导出当前主要导出 ACE-Step BGM 或 Timeline JSON，完整 Beat/SFX 服务端混音仍待补

### 4.3 完整数据流

```
[用户操作]
  ├→ 上传视频 + 选择类型 + 写描述 + 手动标记关键点
  │     ↓
  │   [后端: 视频分析层]
  │     ffmpeg 抽帧 → 帧差计算 → scene_change_candidates
  │     合并用户标记 → video_profile.json
  │     ↓
  │   [后端: AI 编排层]
  │     GLM-4-Flash（或默认模板降级）→ Music Timeline JSON（3-8个segment + ducking schedule）
  │     ↓
  │   [前端: 时间轴编辑]
  │     Canvas 渲染多轨道时间轴
  │     Tone.js 低延迟实时预览 Beat/SFX/Ducking
  │     用户可编辑调整 segment
  │     用户可选择风格模板 restyle
  │     ↓
  │   [后端: 声音生成层]
  │     按 timeline 每个 segment.caption 提交 ACE-Step
  │     ACE-Step 异步生成各段 BGM → ffmpeg concat 拼接
  │     返回完整 BGM 音频 URL
  │     ↓
  │   [前端: 预览]
  │     Tone.Player 加载 BGM + 实时 Beat/SFX/Ducking
  │     多轨混音面板调整各轨音量
  │     ↓
  │   [后端: 导出层]
  │     当前：生成 BGM 转码为 WAV/FLAC/MP3，或直接导出 JSON 配乐方案
  │     目标：ffmpeg amix 混音 BGM + Beat + SFX，并应用 ducking
  │     ↓
  │   [下载 / 发布到社区]
```

## 6. 核心数据模型定义

### 6.1 VideoProject（Module 1）

```sql
-- video_projects 表
CREATE TABLE video_projects (
    id              TEXT PRIMARY KEY,       -- UUID
    user_id         TEXT NOT NULL,          -- 创建者
    title           TEXT,                   -- 用户自定义标题
    video_filename  TEXT NOT NULL,          -- 上传文件名
    video_path      TEXT NOT NULL,          -- 存储路径
    video_type      TEXT NOT NULL,          -- healing_vlog / product_promo / hype_edit / campus_memory / emotional_story / knowledge_edu
    user_description TEXT,                  -- 用户故事描述 ≤500 字
    status          TEXT DEFAULT 'uploaded', -- uploaded / analyzing / profiled / timeline_generated / music_generated / completed
    metadata_json   TEXT,                   -- ffmpeg 提取的元数据(json)
    video_profile   TEXT,                   -- 完整 video_profile.json
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 6.2 MusicTimeline（Module 2）★核心

```sql
-- timelines 表
CREATE TABLE timelines (
    id              TEXT PRIMARY KEY,       -- UUID
    project_id      TEXT NOT NULL UNIQUE,   -- 关联 video_projects.id
    style           TEXT NOT NULL,          -- 风格模板
    bpm             INTEGER NOT NULL,       -- 节拍速度
    key             TEXT NOT NULL,          -- 调式
    duration        REAL NOT NULL,          -- 总时长
    global_caption  TEXT,                   -- 全局音乐描述
    timeline_json   TEXT NOT NULL,          -- 完整 Music Timeline JSON（timeline数组+ducking_schedule）
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (project_id) REFERENCES video_projects(id)
);
```

### 6.3 GenerationTask（Module 3）

```sql
-- generation_tasks 表
CREATE TABLE generation_tasks (
    id              TEXT PRIMARY KEY,       -- UUID
    project_id      TEXT NOT NULL,          -- 关联项目

    ace_task_id     TEXT,                   -- ACE-Step 返回的 task_id

    completed_at    TIMESTAMP,
```

| # | 风险 | 影响等级 | 发生概率 | 应对策略 |
|---|------|---------|---------|---------|
| R1 | **ACE-Step 模型推理失败或超时** | 高 | 中 | ① 设置 120s 超时 + 自动重试 ② 降级方案：改用本地 Tone.js 规则合成简单 BGM（无 ACE 也能演示编排能力） ③ 提前测试 GPU 环境 |
| R2 | **Timeline LLM JSON 输出不稳定或 API key 缺失** | 高 | 中 | ① temperature=0.3 降低随机性 ② 结构校验 + 重试 ③ 无 GLM_API_KEY 时使用默认模板降级 ④ 后续抽象 provider 以支持本地 DeepSeek/Qwen |
| R3 | **Tone.js 多轨同步不同步** | 中 | 低 | ① 统一用 Tone.Transport 调度，不要独立 setTimeout ② schedule 精确到 sample-level ③ 提前充分测试 4 轨并行播放 |
| R4 | **ACE-Step 生成质量不够好** | 中 | 中 | ① 优化 caption prompt（英文、具体、有风格描述） ② 用 cover 模式参考高质量音频 ③ 用 sft 模型替代 turbo（质量更高但更慢） ④ 强调：编排>音质，壁垒在编排 |
| R5 | **视频分析自动检测不准确** | 低 | 高 | ① MVP 以手动标记为主，自动检测仅作辅助建议 ② 明确告诉评委：准确理解需要多模态模型，MVP 阶段先验证编排能力 |
| R6 | **导出 WAV 时间不够** | 低 | 低 | ① 优先 WAV/JSON ② 视频合成标记为 P2 可选 |
| R7 | **被评委认为"套壳"** | 高 | 中 | ① 演示重点突出 Music Timeline（可视化时间轴） ② 强调声音生成引擎可插拔，壁垒在编排 ③ 展示多轨声音设计（不只一条 BGM） |
